Hexo를 이용한 Github Blog 배포

Posted on 2022-09-27

Hexo란?

Static Site Generator중 하나로 Node.js 기반이며 npm을 사용할 수 있기 때문에 쉽게 익힐 수 있는 장점이 있다.

Static Site Generator

기존의 웹페이지는 사용자에게 HTML 파일이 보내지고 그 안에 명시되어 있는 contents들을 표현하기 위해서 server가 요청에
맞는 contents를 database에서 찾고 client에 응답을 보내주는 방식으로 작동이 되었다.
Static Site Generator를 사용한 정적 페이지 구성은 기존의 이런 구조와 다른 방식으로 웹페이지를 구성하는데

Javascript

클라이언트의 요청을 처리한다.

API

Database나 Server 같은 기능은 API가 처리한다.

Markup

Static Site Generator로 Markup 파일을 미리 만들어낸다.(Pre-built Markup)

이렇게 JAM 스택이라고 불리는 형태의 구조로 페이지를 구성한다.

Hexo를 이용한 블로그 생성

Hexo는 Node.js 기반의 SSG이기 때문에 현시간 기준 최소 10.13 버전 이상의 Node.js가 설치되어 있어야 합니다.

Hexo-CLI 설치

Node.js가 설치되어 있다면 npm을 이용해서 Hexo-CLI를 설치해줍니다.

blog 폴더 생성

Hexo-CLI가 설치가 되었다면 이제 blog를 관리할 폴더를 만들어 줍니다.

그리고 해당 폴더로 이동해서

을 해서 블로그 생성에 필요한 패키지들을 설치해줍니다.

Configuration

_config.yml
파일을 사용해서 Configuration을 할 수 있습니다.

title은 블로그 상단의 제목을 의미합니다.
subtitle은 블로그의 부제목을 의미합니다.
description은 블로그 페이지에 대한 설명을 의미하며 HTML의 메타태그에서와 비슷한 의미로 볼 수 있습니다.
author은 작성자 자신의 이름을 의미합니다.

그 외 다양한 환경설정이 가능합니다.

Theme

_config.yml에Extensions를 보면 theme라는 항목이 있습니다.
해당 항목을 이용해서 블로그에 테마를 적용시킬 수 있습니다.

그전에 우선
https://hexo.io/themes/
에서 원하는 테마를 설치합니다.

설치하는 방법은 Hexo가 Node.js 기반이기 때문에 npm을 사용하면 아주 간단합니다.

저는 next라는 테마를 npm을 사용해서 설치를 했습니다.

그 다음 _config.yml의 Extensions의 theme를 next로 바꿔줍니다.
이렇게하면 theme의 적용이 끝납니다.

Blog 배포

Local에서 테스트

Blog를 배포하기 전에 블로그가 잘 생성되었는지 Local에서 확인하는 절차가 필요합니다.

정적 파일들을 먼저 생성해줍니다.

로컬 서버를 구동시킵니다. http://localhost:4000/을 사용합니다.

4000번 포트에 접속하면 블로그가 정상적으로 생성된 것을 확인할 수 있습니다.

Blog 글 작성

Static Site Generator를 사용한 Github 블로그의 글은 일반적으로 Markdown으로 작성합니다.

이렇게 새로운 markdown 파일을 생성해준 다음 해당 파일을 편집해서 글을 작성합니다.

배포

글 작성도 끝났다면 이제 마지막으로 배포를 해줄 차례입니다.

우선 이전에 generate 해뒀던 것을을 정리해줍니다.
배포를 하면 새로 generate가 될 것입니다.
